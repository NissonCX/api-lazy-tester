# 快速开始

5 分钟内让你的 AI Agent 会测试接口。

## 前提条件

- macOS / Linux
- curl 命令可用
- 一个可以测试的 API

## Step 1: 克隆项目

```bash
git clone https://github.com/NissonCX/ai-api-workflow.git
cd ai-api-workflow
```

## Step 2: 准备你的 API

如果你没有可用的 API，可以使用以下公开测试接口：

| 服务 | URL | 说明 |
|------|-----|------|
| JSONPlaceholder | https://jsonplaceholder.typicode.com | 免费 REST API |
| Postman Echo | https://postman-echo.com | 测试请求 |

## Step 3: 运行测试

### 示例 1：GET 请求

```bash
./skill/ai-api-test/scripts/test-api.sh \
  -m GET \
  -u "https://jsonplaceholder.typicode.com/posts/1" \
  -t "Bearer demo-token"
```

### 示例 2：POST 请求

```bash
./skill/ai-api-test/scripts/test-api.sh \
  -m POST \
  -u "https://jsonplaceholder.typicode.com/posts" \
  -t "Bearer demo-token" \
  -b '{"title":"测试","body":"内容","userId":1}'
```

### 示例 3：带超时

```bash
./skill/ai-api-test/scripts/test-api.sh \
  -m GET \
  -u "https://jsonplaceholder.typicode.com/posts/1" \
  -t "Bearer demo-token" \
  -T 10
```

## Step 4: 在 Claude Code 中使用

### 方式一：直接对话

告诉 Claude Code：

```
请测试 GET https://jsonplaceholder.typicode.com/posts/1 接口
Token: Bearer demo-token
```

### 方式二：使用 skill

```
/ai-api-test 测试 GET https://jsonplaceholder.typicode.com/posts/1
Token: Bearer demo-token
```

## 常见问题

### Q: 线上环境怎么测试？

```bash
# 替换 URL 即可
./test-api.sh -m POST \
  -u "https://your-production-api.com/api/login" \
  -t "Bearer your-real-token" \
  -b '{"username":"test"}'
```

### Q: 测试报错 "请求失败或超时"

1. 检查 URL 是否正确
2. 检查网络是否可达
3. 尝试增加超时：`-T 60`

### Q: 状态码 401 未授权

- 检查 token 是否正确
- 检查 token 是否过期

### Q: 想看 simple 格式输出？

```bash
./test-api.sh -m GET -u "https://jsonplaceholder.typicode.com/posts/1" -t "xxx" -s simple
```
