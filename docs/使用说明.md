# 使用说明

本指南面向使用本工作流的 AI Agent 和开发者。

## 一、前置条件

### 1.1 用户需要提供

1. **API Token**：用于认证的 Bearer Token（由用户手动提供）
2. **Base URL**：API 服务的基础地址，例如 `http://localhost:8080`
3. **接口信息**：以下任一方式提供：
   - OpenAPI/Swagger 导出文件（.yaml 或 .json）
   - Controller 源码目录
   - 具体接口路径

### 1.2 环境要求

- AI Agent 需要能够读取项目中的文件
- AI Agent 需要能够发送 HTTP 请求

## 二、快速开始

### 2.1 场景一：测试指定接口

用户提供：
- Token: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
- BaseUrl: `http://localhost:8080`
- 目标接口: `/api/user/login`

AI Agent 执行：
1. 读取 OpenAPI 或 Controller 源码
2. 解析登录接口定义
3. 生成测试用例
4. 执行测试
5. 输出报告

### 2.2 场景二：测试整个模块

用户提供：
- Token: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
- BaseUrl: `http://localhost:8080`
- 目标模块: `用户模块（UserController）`

AI Agent 执行：
1. 查找 UserController 的所有接口
2. 逐个生成并执行测试
3. 输出汇总报告

### 3.3 场景三：测试所有接口

用户提供：
- Token: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
- BaseUrl: `http://localhost:8080`

AI Agent 执行：
1. 读取所有 OpenAPI 定义
2. 扫描所有 Controller
3. 生成完整测试套件
4. 执行并输出报告

## 三、如何提供接口信息

### 3.1 方式一：OpenAPI 文件（推荐）

将 Swagger/OpenAPI 导出文件放到项目中：

```
项目根目录/
  ├── openapi.yaml
  └── api-docs/
      └── swagger.json
```

AI Agent 会自动读取这些文件。

### 3.2 方式二：源码目录

提供 Controller 源码位置：

```
源码目录: src/main/java/com/example/controller
```

AI Agent 会扫描并解析所有 Controller。

### 3.3 方式三：直接指定

用户直接告诉 AI Agent 要测试的接口：

```
测试 POST /api/order/create 接口
```

## 四、测试用例生成规则

### 4.1 正常用例

生成一个最小合法请求：

```json
{
  "username": "a",
  "password": "123456"
}
```

### 4.2 边界用例

根据参数约束生成：

| 约束 | 测试值 |
|------|--------|
| @NotNull | null（不传或传 null） |
| @NotEmpty | ""（空字符串） |
| @Size(min=2, max=10) | "a"（太短）、"12345678901"（太长） |
| @Pattern(regexp="^\\d+$") | "abc"（不符合格式） |
| @Min(0) | -1（小于最小值） |
| @Max(100) | 101（大于最大值） |

### 4.3 异常用例

- 发送空请求体
- 发送错误 Content-Type
- 不带 Authorization 头

## 六、常见问题

### Q1: 没有 OpenAPI 文件怎么办？

AI Agent 会尝试从 Controller 源码中提取信息。如果源码也没有，则无法执行测试。

### Q2: Token 过期怎么办？

用户需要提供新的 Token。AI Agent 不会自动刷新 token。

### Q3: 接口返回 500 错误怎么办？

这通常表示服务端问题。AI Agent 会记录错误信息并在报告中标注。

### Q4: 可以测试需要文件上传的接口吗？

可以，但需要用户额外提供测试文件路径。

### Q5: 支持 WebSocket 测试吗？

当前版本不支持。V2 会考虑支持。

## 七、最佳实践

1. **先测读接口**：先用 GET 请求熟悉接口
2. **再测写接口**：确认读取正常后再测试创建/更新
3. **关注状态码**：验证响应状态码是否符合预期
4. **检查响应体**：验证返回的数据结构是否正确
5. **记录错误**：遇到问题保留请求/响应以便排查
