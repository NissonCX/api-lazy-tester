# 设计边界与风险

本文档明确说明本项目的设计边界和风险控制原则。

## 一、设计边界

### 1.1 核心目标

- ✅ 定义 AI Agent 执行接口测试的工作流规范
- ✅ 提供可复用的测试用例生成规则
- ✅ 支持 OpenAPI 和源码两种信息源
- ✅ 输出结构化测试报告

### 1.2 非目标

- ❌ 不实现自动获取 token 功能
- ❌ 不实现复杂的测试执行引擎
- ❌ 不实现测试用例管理界面
- ❌ 不实现持续集成支持
- ❌ 不实现团队协作功能
- ❌ 不实现测试数据生成器

### 1.3 适用范围

| 场景 | 是否适用 |
|------|----------|
| 个人开发者日常接口验证 | ✅ 适用 |
| AI Agent 辅助测试 | ✅ 适用 |
| 快速冒烟测试 | ✅ 适用 |
| 接口契约验证 | ✅ 适用 |
| 复杂测试场景 | ❌ 不适用 |
| 团队协作测试 | ❌ 不适用 |
| 持续集成 | ❌ 不适用 |
| 压力测试 | ❌ 不适用 |

## 二、风险控制

### 2.1 数据安全

#### 原则

- **不存储 token**：AI Agent 使用完 token 后不应持久化存储
- **不传输敏感数据**：测试请求中的敏感信息应被记录但不公开
- **本地处理优先**：测试过程在本地执行，不上传到第三方服务

#### 风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| Token 泄露 | 高 | 不存储，运行时使用 |
| 测试数据泄露 | 中 | 报告不含敏感信息 |
| 请求日志泄露 | 低 | 本地记录，不上传 |

### 2.2 环境安全

#### 原则

- **默认本地优先**：优先测试本地环境（localhost）
- **生产需确认**：测试生产环境需要明确确认
- **只读操作优先**：优先执行 GET 等只读操作

#### 风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 误操作生产环境 | 高 | 明确提示，强制确认 |
| 意外修改数据 | 中 | 默认只读操作 |
| 请求风暴 | 低 | 单次请求，不并发 |

### 2.3 操作安全

#### 高风险操作列表

以下操作被定义为高风险，默认禁止：

1. **DELETE 请求**：可能永久删除数据
2. **批量操作**：可能影响多条记录
3. **导入/导出**：可能影响大量数据
4. **配置修改**：可能影响系统运行
5. **支付相关**：可能产生真实交易
6. **用户数据操作**：可能影响真实用户

#### 确认流程

执行高风险操作前，必须：

1. 停止执行
2. 向用户说明风险
3. 请求明确确认（yes/no）
4. 确认后执行，否则跳过

## 三、限制说明

### 3.1 功能限制

| 限制 | 说明 |
|------|------|
| 并发测试 | 不支持多并发请求 |
| 定时任务 | 不支持定时执行 |
| 循环测试 | 不支持循环执行 |
| 依赖测试 | 不支持测试用例间依赖 |
| 状态保持 | 不支持 Cookie/Session 保持 |

### 3.2 参数限制

| 限制 | 说明 |
|------|------|
| 文件上传 | 仅支持简单文件上传 |
| 复杂类型 | 不支持复杂的自定义类型 |
| 嵌套对象 | 嵌套超过 3 层可能解析失败 |
| 循环引用 | 不支持循环引用的 DTO |

### 3.3 环境限制

| 限制 | 说明 |
|------|------|
| HTTPS 证书 | 不验证自签名证书 |
| 认证方式 | 仅支持 Bearer Token |
| 代理 | 不支持代理 |
| 超时 | 不支持自定义超时 |

## 四、异常处理

### 4.1 网络异常

- 请求超时：记录超时错误，继续执行下一个用例
- 连接失败：记录连接错误，标记该接口为失败

### 4.2 响应异常

- 非 JSON 响应：记录原始响应内容
- 异常状态码：记录状态码和错误信息

### 4.3 解析异常

- 无法解析的参数：跳过该参数，生成通用测试
- 无法解析的响应：记录原始响应

4. **数据备份**：执行修改类操作前，请确保已备份数据
