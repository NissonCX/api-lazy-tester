# 项目介绍

## 背景

在日常开发中，Java 后端开发者经常需要测试自己开发的 API 接口。传统方式有：
- 使用 Postman/Insomnia 手动创建请求
- 编写单元测试
- 使用 Swagger UI 手动测试

但这些方式都有一些痛点：
1. 手动创建请求耗时，尤其是复杂 DTO
2. 单元测试需要额外编写代码
3. Swagger UI 不适合批量测试

AI 的出现带来了新的可能：AI 能够理解代码结构，能否让 AI 自动完成接口测试？

## 问题

市面上的 AI 代码助手（Claude Code、Codex 等）虽然能理解代码，但没有一套系统化的接口测试工作流。它们不知道：
- 如何读取 OpenAPI 定义
- 如何提取 DTO 结构
- 如何生成合理的测试用例
- 如何判断测试结果

## 解决方案

本项目定义了一套面向 AI Agent 的接口测试工作流规范，让 AI Agent 能够：

1. **读取接口定义**：从 OpenAPI/Swagger 或源码中获取接口信息
2. **理解数据模型**：解析 DTO、Bean Validation 注解
3. **生成测试用例**：根据接口定义生成最小合法请求和负例
4. **执行测试**：调用实际接口并验证响应
5. **输出报告**：生成结构化的测试结果

## 核心特点

### 1. 规范优先

本项目不实现复杂的测试执行引擎，而是定义工作流规范。任何支持这些规范的 AI 工具都可以使用。

### 2. token 由用户提供

考虑到安全性，本项目不实现自动获取 token 的功能。token 由用户在测试时手动提供。

### 3. 安全优先

- 默认不执行高风险写操作
- 批量删除需要明确确认
- 禁止生产环境写操作

### 4. 可扩展

工作流规范是开放的，可以根据需要扩展和定制。

## 与现有工具的区别

| 特性 | 本项目 | Postman | Swagger UI | 自动化测试框架 |
|------|--------|---------|------------|----------------|
| AI 驱动 | 是 | 否 | 否 | 否 |
| 自动生成请求 | 是 | 手动 | 手动 | 需编码 |
| token 管理 | 用户提供 | 支持 | 支持 | 需编码 |
| 学习成本 | 低 | 中 | 低 | 高 |
| 适用场景 | AI Agent 辅助测试 | 手动测试 | 手动测试 | 持续集成 |

## 适用人群

- 个人开发者：日常接口快速验证
- AI 工具用户：希望 AI 辅助测试接口
- 团队：建立内部接口测试规范

## 不适用场景

- 需要复杂测试报告
- 需要持续集成
- 需要团队协作测试管理
- 需要自动获取 token

这些场景更适合使用专业的测试工具如 Postman、Jmeter 等。
