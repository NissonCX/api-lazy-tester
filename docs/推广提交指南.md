## 5. GitHub Topics

确保项目包含以下 topics：
- api-testing
- claude-code
- ai-agent
- testing-tools

在 GitHub 项目设置中添加：Settings → General → Topics

不要把 topics 或仓库简介写进 `.github/README.md`。
GitHub 会优先展示 `.github/README.md`，这会覆盖仓库首页本应显示的根目录 `README.md`。

## 提交模板

### Issue 模板（推荐使用）

```markdown
## 项目名称
api-lazy-tester

## 项目描述
AI Agent 接口测试工作流，让 AI 自动测试后端 API 接口

## 项目地址
https://github.com/NissonCX/api-lazy-tester

## 相关标签
api-testing, claude-code, ai-agent
```

### Pull Request 模板（Awesome 列表）

```markdown
## 添加 api-lazy-tester

- **项目名称**: api-lazy-tester
- **描述**: 接口写完了，测试的事交给 AI
- **GitHub**: https://github.com/NissonCX/api-lazy-tester
- **免费/开源**: 是
```

## 检查清单

- [x] README.md 内容完整
- [x] 有使用示例

## 优先顺序

1. **高优先级**: Awesome 列表（曝光大）
2. **中优先级**: ToolHub、OpenTools（精准开发者）
3. **低优先级**: 社区 issues、社交媒体（需要持续运营）
