# 更多使用示例

本文档提供丰富的使用示例。

## 基础示例

### 1. GET 请求（获取数据）

```bash
./test-api.sh -m GET \
  -u "https://jsonplaceholder.typicode.com/users/1" \
  -t "Bearer demo-token"
```

### 2. POST 请求（创建数据）

```bash
./test-api.sh -m POST \
  -u "https://jsonplaceholder.typicode.com/posts" \
  -t "Bearer demo-token" \
  -b '{"title":"我的文章","body":"内容","userId":1}'
```

### 3. PUT 请求（更新数据）

```bash
./test-api.sh -m PUT \
  -u "https://jsonplaceholder.typicode.com/posts/1" \
  -t "Bearer demo-token" \
  -b '{"id":1,"title":"更新后的标题","body":"更新后的内容","userId":1}'
```

### 4. DELETE 请求（删除数据）

⚠️ 删除操作需要用户确认

```bash
./test-api.sh -m DELETE \
  -u "https://jsonplaceholder.typicode.com/posts/1" \
  -t "Bearer demo-token"
```

## 进阶示例

### 5. 带查询参数

```bash
# URL 中直接包含查询参数
./test-api.sh -m GET \
  -u "https://jsonplaceholder.typicode.com/posts?userId=1" \
  -t "Bearer demo-token"
```

### 6. 带自定义请求头

```bash
./test-api.sh -m POST \
  -u "https://api.example.com/api/data" \
  -t "Bearer demo-token" \
  -H "X-Request-ID: test-001" \
  -H "X-Custom-Header: value" \
  -b '{"data":"test"}'
```

### 7. 增加超时

```bash
# 慢接口，增加到 60 秒
./test-api.sh -m GET \
  -u "https://slow-api.example.com/heavy-endpoint" \
  -t "Bearer demo-token" \
  -T 60
```

### 8. 简单输出格式

```bash
# 适合调试，快速查看结果
./test-api.sh -m GET \
  -u "https://jsonplaceholder.typicode.com/posts/1" \
  -t "Bearer demo-token" \
  -s simple
```

## 完整工作流示例

### 场景：测试用户登录接口

假设你有一个登录接口：

```
POST http://localhost:8080/api/auth/login
Content-Type: application/json
Body: {"username": "test", "password": "123456"}
```

#### Step 1: 测试正常登录

```bash
./test-api.sh -m POST \
  -u "http://localhost:8080/api/auth/login" \
  -t "Bearer test-token" \
  -b '{"username":"test","password":"123456"}'
```

预期响应：
```json
{
  "response": {"status": 200, "body": {"token": "xxx", "userId": 1}},
  "result": "通过"
}
```

#### Step 2: 测试用户名为空

```bash
./test-api.sh -m POST \
  -u "http://localhost:8080/api/auth/login" \
  -t "Bearer test-token" \
  -b '{"username":"","password":"123456"}'
```

预期响应：
```json
{
  "response": {"status": 400, "body": {"error": "用户名不能为空"}},
  "result": "通过"
}
```

#### Step 3: 测试密码错误

```bash
./test-api.sh -m POST \
  -u "http://localhost:8080/api/auth/login" \
  -t "Bearer test-token" \
  -b '{"username":"test","password":"wrong-password"}'
```

预期响应：
```json
{
  "response": {"status": 401, "body": {"error": "用户名或密码错误"}},
  "result": "通过"
}
```

## 在 Claude Code 中使用

### 方式一：告诉 AI 你要测试什么

```
请测试 GET https://jsonplaceholder.typicode.com/posts/1
Token: Bearer demo-token
```

AI 会：
1. 读取 skill 了解工作流
2. 调用脚本执行测试
3. 输出测试结果

### 方式二：使用 skill 命令

```
/ai-api-test GET https://jsonplaceholder.typicode.com/posts/1 Bearer demo-token
```

### 方式三：批量测试

```
请测试用户模块的所有接口
Token: Bearer xxx
BaseUrl: http://localhost:8080
Controller: UserController
```

AI 会：
1. 查找 UserController 的所有接口
2. 逐个生成测试用例
3. 执行并汇总报告

  -u "http://localhost:8080/api/test" \
